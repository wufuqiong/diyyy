# 百数板拼图填空练习题生成器 — 设计文档 v2.0

> **产品名称**：Hundred Chart Puzzle Worksheet Maker
> **目标用户**：小学一年级数学教师、家长
> **核心目标**：生成"百数板拼图块填空"练习题——每道题是百数板中一块随机形状的连通格子，给出部分数字，让学生利用 ±1（左右）/ ±10（上下）规律填写其余格子。

---

## 一、题型定义

### 1.1 核心概念

每道题是百数板中的一块**任意形状的连通格子拼图**，满足：

1. **连通性**：所有格子通过上下左右相邻形成一个整体，不允许对角线连接，不允许孤立格子。
2. **数值合法性**：每个格子的数值等于其在标准百数板（1–100，10列×10行）中的真实位置数字。因此，同行相邻格子差值为 1，上下相邻格子差值为 10。
3. **可推导性**：给出的已知格子足以让学生唯一推导出所有留空格子的数值。

```
百数板坐标规则：
  数值 = (行 - 1) × 10 + 列
  行 ∈ [1, 10]，列 ∈ [1, 10]

示例：
  位置 (3, 5) → 数值 25
  位置 (7, 2) → 数值 62
  左邻格 = 数值 - 1，右邻格 = 数值 + 1
  上邻格 = 数值 - 10，下邻格 = 数值 + 10
```

### 1.2 难度档位

| 档位 | 格子数 | 形状 | 已知格数 | 留空格数 | 适用场景 |
|------|--------|------|---------|---------|---------|
| **简单（Easy）** | 5 | 固定十字形（+） | 2–3 个 | 2–3 个 | 初次接触，形状规律易理解 |
| **中等（Medium）** | 5–6 | 随机连通形状 | 2–3 个 | 2–4 个 | 标准练习，需识别形状方向 |
| **困难（Hard）** | 5–9 | 随机连通形状 | 仅 1 个 | 4–8 个 | 高阶推理，需链式推导 |

**简单档的十字形固定结构**（中心格 + 上下左右各一格）：
```
      [ 中心-10 ]
[ 中心-1 ] [中心] [ 中心+1 ]
      [ 中心+10 ]
```

**中等 / 困难档的随机形状举例**（括号内为格子数值，★ 为已知格）：

```
L形(5格)         S形(6格)         T形(6格)         不规则(7格)
31  32            54  55★          22  __  __       13  14
41★             63★  64            33★              23★  24  25
51  52           73   74            43   44              34  35
```

### 1.3 已知格的选取原则

已知格的位置直接决定题目是否有唯一解，选取时优先考虑：

1. **锚定效应**：优先选择与多个留空格相邻的"枢纽格"——一个已知格能约束更多未知格。
2. **角落格**：形状边缘只有 1–2 个邻居的格子，已知后可唯一确定邻格数值。
3. **唯一解验证**：生成后算法自动模拟学生推导过程，验证从已知格出发能推导出全部留空格；若不能，则重新选取已知格或调整形状。
4. **避免冗余**：不选互为邻格的两个格子都作为已知格（信息重叠，降低挑战性）。

### 3.2 单道题渲染规范

每道题在一个**边界框（bounding box）**内渲染，bounding box 为该题所有格子的最小外接矩形。

```
bounding box（虚线，不打印）
┌─────────────────┐
│      ┌───┐      │
│      │35 │      │  ← 已知格：数字加粗，深色
│  ┌───┼───┼───┐  │
│  │__ │44 │__ │  │  ← 已知格（44），留空格（__）
│  └───┴───┴───┘  │
│      ┌───┐      │
│      │__ │      │  ← 留空格
│      └───┘      │
└─────────────────┘
题号 ① 显示在 bounding box 左上角外侧
```

**格子渲染细节**：
- **已知格**：白色填充，黑色粗边框，数字居中加粗，字色 #222。
- **留空格**：白色填充，黑色边框（同粗细），内容根据"空格样式"设置。
- **题号**：灰色小字，显示在 bounding box 左上角，如 `①`。
- **格子间距**：格子紧邻，无额外间距（拼图感）。
- **bounding box 不渲染**：仅用于内部布局计算。

## 五、题目生成算法

### 5.1 整体流程

```
对每道题：
  1. 选取锚点格（anchor cell）
  2. 用随机游走生成连通形状
  3. 验证形状合法性
  4. 选取已知格
  5. 验证唯一解
  6. 渲染输出
```

### 5.2 Step 1：选取锚点格

```javascript
function pickAnchor(config, usedAnchors, seed) {
  // 在 [minAnchor, maxAnchor] 范围内随机选一个未使用数值
  // 转换为 (row, col)：row = ceil(n/10), col = n - (row-1)*10
  // 确保锚点格的四个方向中至少有 (targetCellCount - 1) 个合法邻格可供扩展
}
```

### 5.3 Step 2：随机游走生成形状

```javascript
function generateShape(anchor, targetCount, shapeBias, seed) {
  const cells = new Set([anchor]);        // 从锚点出发
  const frontier = getValidNeighbors(anchor); // 合法邻格列表

  while (cells.size < targetCount && frontier.length > 0) {
    // 根据 shapeBias 调整方向权重：
    //   Row-heavy → 左右方向权重 × 2
    //   Column-heavy → 上下方向权重 × 2
    //   Irregular → 优先选择使形状更不规则的方向
    const next = weightedRandom(frontier, seed);
    cells.add(next);
    // 将 next 的未加入邻格添加到 frontier（去重）
    addNewNeighbors(frontier, next, cells);
  }

  if (cells.size < targetCount) return null; // 空间不足，重试
  return cells;
}

function getValidNeighbors(cell) {
  // 返回上下左右四个方向中在百数板范围内的格子
  // 特别处理行边界：第1列无左邻，第10列无右邻
  const { row, col } = toRowCol(cell);
  const neighbors = [];
  if (row > 1)  neighbors.push(cell - 10); // 上
  if (row < 10) neighbors.push(cell + 10); // 下
  if (col > 1)  neighbors.push(cell - 1);  // 左
  if (col < 10) neighbors.push(cell + 1);  // 右
  return neighbors;
}
```

### 5.4 Step 3：验证形状合法性

```javascript
function validateShape(cells) {
  // 1. 连通性检验：从任意一格 BFS/DFS，能到达所有格子
  // 2. 数值合法性：所有格子值在 [1, 100]，行列坐标一致
  // 3. 形状唯一性（若开启禁止重复）：
  //    将形状归一化（平移到左上角为原点），与已生成形状集比较
  return isConnected(cells) && allValuesValid(cells);
}
```

### 5.5 Step 4：选取已知格

```javascript
function selectHints(cells, hintCount, seed) {
  // 计算每个格子的"锚定权重"：
  //   权重 = 该格子在形状内的邻格数（度数）
  //   度数高的格子更适合作为已知格（信息扩散效率高）
  const weights = computeAnchorWeights(cells);

  // 按权重从高到低排序，选取 hintCount 个格子作为已知格
  // 加入随机扰动（乘以随机因子），避免总是选相同位置
  return weightedSample(cells, weights, hintCount, seed);
}
```

### 5.6 Step 5：唯一解验证

```javascript
function validateUniqueSolution(cells, hints) {
  // 模拟学生推导过程（BFS）：
  //   已知队列 = hints 集合
  //   while 队列非空：
  //     取出已知格 c
  //     对 c 的每个邻格 n（在 cells 内）：
  //       若 n 未确定，则 n 的值唯一确定 → 加入队列
  //   若最终所有 cells 都确定 → 有唯一解 ✓
  //   否则 → 无唯一解，需重新选取已知格或调整形状
  const resolved = propagate(cells, hints);
  return resolved.size === cells.size;
}
```

### 5.7 重试机制

```javascript
const MAX_RETRIES = 50;
for (let i = 0; i < MAX_RETRIES; i++) {
  const anchor = pickAnchor(config, usedAnchors, seed + i);
  const shape  = generateShape(anchor, targetCount, config.shapeBias, seed + i);
  if (!shape || !validateShape(shape)) continue;

  const hints  = selectHints(shape, hintCount, seed + i);
  if (validateUniqueSolution(shape, hints)) {
    return { shape, hints }; // 成功
  }
}
throw new Error('生成失败：请调整参数（如扩大数字范围或降低格子数量）');
```

## 七、技术实现要点

| 方面 | 方案 |
|------|------|
| **框架** | 纯 HTML + Vanilla JS（单文件，无构建工具，直接打开即用）|
| **随机种子** | 自实现 LCG 伪随机，相同 seed 完全复现 |
| **拼图渲染** | 每道题用 CSS Grid（3–5 列），空白占位格用 `visibility: hidden` 实现任意形状 |
| **唯一解验证** | BFS 约束传播（constraint propagation），O(n) 复杂度 |
| **打印** | `@media print` CSS + `window.print()` |
| **PDF 导出（可选增强）** | `html2canvas` + `jsPDF` |

## 九、后续可扩展题型

| 题型 | 说明 |
|------|------|
| **大步长拼图** | 改为 ±2 / ±20，适用更大数字范围（1–200 等）|
| **斜向拼图** | 加入对角线方向（±9 / ±11），考查斜向规律 |
| **连锁拼图** | 多个拼图块共用格子，形成更大的连接结构 |
| **反推模式** | 只给出留空格的数值（不含位置），推断整块形状 |
| **填色分区** | 多块拼图叠加在同一个百数板上，用不同颜色区分，综合练习 |
