# 待跟踪问题：Char Maze PHRASE 模式静默跳过无法放置的词语

**发现时间/方式**：2026-06-24，编写 Layer 1 测试时，确认 `generatePhraseMazePath` 真实行为后发现（非用户报告，是代码审查过程中的衍生发现）。

**现象**：PHRASE 模式下，如果某个词语因为长度或位置限制无法放入迷宫网格，`generatePhraseMazePath` 仅执行 `console.warn`，跳过该词语，不抛异常，不返回任何结构化信号。最终用户拿到的迷宫页面里，这个词语会"凭空消失"，没有任何提示。

**复现条件**：在较小的 grid 上（如 8×8）放置多个较长词语（如每页 5 个 3-4 字词组），部分词语因 grid 空间不足无法放置。

**与既往发现的关系**：这是同一类"静默失败"问题的第三个已知实例：
1. 识字迷宫切换 Mode 清空内容时无确认提示（已修复：添加确认对话框）
2. hundred-chart 数值越界静默 clamp（已确认，待处理）
3. Char Maze PHRASE 模式词语静默跳过（当前记录）

**建议的修复方向（待产品确认，不在本次任务中实施）**：
- 方案 A：参照 SENTENCE 模式的处理方式，抛出 Error，由 `Workbench` 转化为可见 Alert
- 方案 B：返回结果中增加 `unplacedWords` 字段（参照 word-search 工具已有的同名设计），由 Preview 组件检测并显示提示

两种方案的选择需要产品侧决定。

**当前测试覆盖状态**：已在 `TEST.md` Charmaze 测试章节标注为已知局限。测试断言"不抛异常"，未断言"用户能感知到"（因为目前没有可感知的信号可断言）。
